Story

The story.json file is the entry point of the story. For stories without choice, the entire story can be contained in this file.

The story file should contain an array with each of the steps of the Fable Story, each step will be defined as an Object containing all the required information for that step.

The Fable Story player will reproduce each step after a tap on the screen on the same order as defined on this file.

Step#

Each step could behave differently depending on the step type, and the required information will change based on that. Next is the minimum required information for each step.

  • type: definition of the step type
  • version: [Optional] defaulting to 1, provides a way to improve step functionality without redefining the entire story spec.

Step Types#

Speech Bubble#

This represents a conversation bubble in which one of the characters is saying something. This bubbles must be assigned to a character.

  • text: Text that the bubble will contain
  • smile: [Optional] file name of the smile under assets/smiles folder. If present, text will be ignored making it optional.
  • character: KeyName of the character

The bubble will adjust color and alignment based on character definition.

[  ...,  {    "type": "speech",    "text": "I'm Iron Man",    "character": "tonyStark",  },  ...  {    "type": "speech",    "smile": "angry.png",    "character": "pepperPotts",  },  ...]

Story Development Bubble#

This represents a ory development bubble. It could be assign to a character or not.

  • text: Text that the bubble will contain
  • smile: [Optional] file name of the smile under assets/smiles folder. If present, text will be ignored making it optional.
  • character: [Optional] KeyName of the character
  • align: [Optional] {def: center} The alignment of the bubble
  • color: [Optional] {def: none} The color of the bubble

This bubbles will be center aligned by default if no character and no align is provided. If a character is provided, it will use the character's color and alignment ignoring color and align definitions.

[  ...,  {    "type": "text",    "text": "The crowd was astonished by the confession",    "color": "blue"  },  ...]

Media#

This bubble represent a media type of image, video or voice. The assets should be present under the assets/images, assets/videos or assets/voice folders respectively.

This bubbles will be center aligned by default if no character and no align is provided. If a character is provided, it will use the character's color and alignment ignoring color and align definitions.

  • media: One of image, video or voice
  • path: Path of the file under the corresponding folder
  • character: [Optional] KeyName of the character
  • align: [Optional] {def: center} The alignment of the bubble
  • color: [Optional] {def: none} The color of the bubble
[  ...,  {    "type": "media",    "media": "video",    "path": "flying.mp4",    "color": "blue"  },  ...  {    "type": "media",    "media": "voice",    "path": "speech.mp3",    "character": "tonyStark"  },  ...]

Choice#

This will give the reader the possibility to make a choice and choose it's own story path.

By providing this step type, we cna branch the main story into different sub-stories depending on user choice. Each sub-story will be defined on a separate file, with a similar file format under the choices folder.

Each choice will be represent as a button inside of the bubble holding a descriptive text.

  • text: Text that the bubble will contain
  • choices: Array of choice objects

Choices will be shown in the same order of the array order.

Choice Object#

A Choice object holds the choice configuration.

  • text: The text that the choice button will contain
  • branch: filename of the branch that will be loaded when the user pick this choice under the choices folder
[  ...,  {    "type": "choice",    "text": "Tony is about to propose Pepper...",    "choices": [      {        "text": "Tony bends on his knees and proposes Pepper",        "branch": "proposal.json"      },      {        "text": "Tony got distracted by the waitress and get slapped",        "branch": "angry-pepper.json"      },    ]  },  ...]

Navigation#

This step represents a direct navigation to another branch. The player will navigate without user intervention and display the first step of the navigated branch after the user tap.

  • branch: filename of the branch that will be loaded under the choices folder
[  ...,  {    "type": "nav",    "branch": "main-thread-1.json"  },  ...]
